http 流程的生命周期

在开发模块前我们需要了解http流程的生命周期，然后确定我们需要扩展的功能在哪个阶段，最后才能在该阶段介入我们扩展的功能。

nginx 的http流程有11个阶段，每个阶段都可以认为是单独的模块在负责对应的职责，每个阶段的处理可能不止一次，可能会发生循环作用，这个完全由每个模块来决定对应的后续行为，如下为默认基础的11个生命周期:
